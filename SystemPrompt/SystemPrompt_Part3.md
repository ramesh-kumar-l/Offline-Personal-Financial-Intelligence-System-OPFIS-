# ====================================================================
# SYSTEM PROMPT
#
# PART 3
#
# Premium Product Design System
#
# This section defines how every screen of OPFIS must be designed.
#
# Every UI must feel intentional, premium, trustworthy and timeless.
#
# UI is part of the product.
#
# Users judge engineering quality through UI quality.
# ====================================================================

# DESIGN PHILOSOPHY

The design language should communicate

Trust

Clarity

Calmness

Precision

Confidence

Privacy

Professionalism

Never design screens that look

Flashy

Playful

Crypto-themed

Gaming inspired

Marketing heavy

Social media inspired

OPFIS is a professional productivity application.

----------------------------------------------------------------

# DESIGN INSPIRATION

Take inspiration from

Apple Health

Apple Wallet

Apple Passwords

Apple Journal

Google Wallet

Google Photos

Notion

Linear

Stripe Dashboard

Raycast

Arc Browser

The goal is NOT to copy.

The goal is to achieve

clarity

consistency

craftsmanship

----------------------------------------------------------------

# FRONTEND DESIGN SKILL

Whenever UI work is requested

Always use

/frontend-design

before implementation.

The frontend should never be generated
without thoughtful design.

Produce

Wireframes

Component hierarchy

Navigation flow

State diagrams

Interaction model

Accessibility considerations

before writing code.

----------------------------------------------------------------

# DESIGN PRINCIPLES

Every screen must answer

What is happening?

Where am I?

What can I do?

What changed?

What is important?

What requires attention?

Never make users guess.

----------------------------------------------------------------

# TRUST FIRST DESIGN

Financial software must feel trustworthy.

Users should immediately feel

My data is safe.

My data is local.

Nothing hidden is happening.

The interface should communicate

confidence

not excitement.

----------------------------------------------------------------

# VISUAL LANGUAGE

Prefer

Whitespace

Alignment

Hierarchy

Readable typography

Meaningful colors

Simple layouts

Avoid

Visual clutter

Heavy gradients

Large shadows

Excessive animations

Bright saturated colors

Marketing effects

----------------------------------------------------------------

# INFORMATION DENSITY

Information should be

Dense enough for professionals

Simple enough for beginners

Use progressive disclosure.

Do not overwhelm users.

----------------------------------------------------------------

# NAVIGATION

Navigation must remain shallow.

Maximum

Three levels deep.

Primary navigation should expose

Dashboard

Accounts

Assets

Transactions

Goals

Documents

Insights

Search

Settings

Users should never become lost.

----------------------------------------------------------------

# HOME DASHBOARD

The dashboard should answer

What is my current financial health?

Net Worth

Cash

Investments

Liabilities

Monthly Cash Flow

Goal Progress

Recent Activity

Insights

Today's Recommendations

No unnecessary charts.

Every widget must justify its existence.

----------------------------------------------------------------

# SEARCH

Search should be globally accessible.

One shortcut.

One search box.

Everything searchable.

Examples

Assets

Transactions

Loans

Goals

Documents

Receipts

Statements

People

Institutions

Categories

Tags

Search should feel instant.

----------------------------------------------------------------

# AI ASSISTANT

AI should feel

Helpful

Calm

Explainable

Never pretend certainty.

Always explain reasoning.

Allow users to inspect
the underlying financial records.

Never fabricate financial information.

----------------------------------------------------------------

# COLOR SYSTEM

Colors communicate meaning.

Primary

Professional Blue

Success

Green

Warning

Amber

Error

Red

Information

Neutral Blue

Background

Neutral Gray

Never rely on color alone.

Every color meaning
must also include

icon

text

shape

----------------------------------------------------------------

# TYPOGRAPHY

Typography is hierarchy.

Use

Display

Heading

Title

Body

Caption

Mono

Never use more than
one font family.

Numbers should align correctly.

Financial values
must always be highly readable.

----------------------------------------------------------------

# ICONOGRAPHY

Icons support understanding.

Icons never replace text.

Icons should remain

Minimal

Consistent

Professional

Never decorative.

----------------------------------------------------------------

# COMPONENT LIBRARY

Every UI component must be reusable.

Buttons

Cards

Lists

Charts

Dialogs

Bottom Sheets

Snackbars

Search

Filters

Tables

Forms

Selectors

Tags

Badges

Timeline

Financial Widgets

Never duplicate components.

----------------------------------------------------------------

# CARD DESIGN

Cards should communicate

One responsibility.

Avoid giant cards
containing multiple unrelated concepts.

Cards should support

Expansion

Actions

Context

Accessibility

Loading

Empty State

----------------------------------------------------------------

# TABLES

Financial applications require tables.

Tables should support

Sorting

Filtering

Searching

Grouping

Sticky headers

Responsive layout

Column visibility

Export

Tables should remain usable
for thousands of records.

----------------------------------------------------------------

# CHARTS

Charts exist

to explain.

Not decorate.

Preferred charts

Net Worth Trend

Cash Flow

Asset Allocation

Category Breakdown

Goal Progress

Portfolio Performance

Timeline

Avoid

3D Charts

Fancy animations

Unnecessary gradients

----------------------------------------------------------------

# MOTION

Motion should communicate

State change

Navigation

Completion

Loading

Nothing else.

Animations must remain

Subtle

Fast

Purposeful

Avoid decorative motion.

----------------------------------------------------------------

# LOADING STATES

Never show blank screens.

Every loading state
should communicate progress.

Skeleton screens preferred.

Avoid infinite spinners.

----------------------------------------------------------------

# EMPTY STATES

Empty states should educate.

Explain

Why empty

What user can do

How to start

Never display

"No Data"

alone.

----------------------------------------------------------------

# ERROR STATES

Every error should explain

What happened

Why

Recovery

Retry

Support if needed

Never expose stack traces.

Never blame the user.

----------------------------------------------------------------

# ACCESSIBILITY

Accessibility is mandatory.

Support

Screen Readers

Dynamic Font Sizes

Keyboard Navigation

High Contrast

Color Blind Users

Touch Targets

Focus Indicators

WCAG AA minimum.

----------------------------------------------------------------

# RESPONSIVE DESIGN

Design should adapt

Phone

Tablet

Desktop

Foldables

Large Screens

Never create separate products.

Use adaptive layouts.

----------------------------------------------------------------

# DARK MODE

Dark mode is first-class.

Not an afterthought.

Every component

must be validated
in both themes.

----------------------------------------------------------------

# FORMS

Financial forms should

Reduce typing

Auto-complete

Validate immediately

Save drafts

Support undo

Never surprise users.

----------------------------------------------------------------

# CONFIRMATION DIALOGS

Only confirm

Destructive actions

Data deletion

Reset

Backup overwrite

Avoid unnecessary confirmations.

----------------------------------------------------------------

# SETTINGS

Settings should remain minimal.

Organize

Security

Backup

Appearance

AI

Import

Export

Advanced

No clutter.

----------------------------------------------------------------

# TRUST INDICATORS

The application should continuously
communicate trust.

Examples

Database encrypted

Offline Mode

Last Backup

AI Running Locally

No Cloud Connected

These should reassure,
not alarm.

----------------------------------------------------------------

# PREMIUM FEEL

Every interaction should feel

Predictable

Responsive

Refined

Professional

No abrupt transitions.

No inconsistent spacing.

No inconsistent typography.

No inconsistent terminology.

----------------------------------------------------------------

# UI REVIEW CHECKLIST

Before implementing any screen

verify

Clear hierarchy

Consistent spacing

Accessibility

Responsive

Keyboard support

Dark mode

Loading state

Empty state

Error state

Visual consistency

Trust indicators

If any answer is NO

Redesign before implementation.

----------------------------------------------------------------

# FRONTEND COMPLETION CONTRACT

A screen is complete only if

Design approved

Responsive

Accessible

Reusable components

Tests added

Dark mode verified

Documentation updated

Memory updated

Otherwise

The screen is incomplete.
