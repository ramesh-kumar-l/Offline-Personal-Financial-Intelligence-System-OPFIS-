# OPFIS Demo Walkthrough

A scripted, ~10-minute walkthrough of every screen, run entirely
offline against a fresh local database. No accounts, no network
connection, and no seed-data script are required or provided - OPFIS
starts with zero data by design, and this script creates just enough
as it goes.

Start the app first:

```sh
./gradlew :composeApp:run
```

The app opens to the **lock screen**. On Desktop, tap **Confirm to
unlock** (there is no OS-uniform biometric API on the JVM, so Desktop
always uses this manual fallback - this is documented behavior, not a
bug). On Android, use your device's biometric prompt.

## 1. Dashboard

You land on **Dashboard**. With no data yet, Net Worth reads 0 and the
Asset Allocation/Cash Flow charts are empty - that's expected on first
run. This screen recomputes live from whatever you enter in the next
steps, so come back to it at the end.

## 2. Create some data (via Search's timeline / Vault / Memory forms)

OPFIS doesn't have a single "add data" screen; each entity type is
created from the screen it belongs to. For this demo:

1. Go to **Vault** and import one PDF or image file (a sample receipt,
   invoice, or statement works). Watch the extracted text appear once
   OCR/PDF-text extraction finishes - it happens locally, no upload.
2. Go to **Memory** and record one note (e.g. "Started this demo") -
   this is the hand-recorded financial timeline, not auto-generated.
3. Go to **Search**, type a keyword from the document you just
   imported (part of its filename or extracted text). The result
   should appear instantly (SQLite FTS5, no debounce delay needed to
   feel it) - this proves Vault's import wired straight into the
   global search index.

## 3. Search & Timeline

Still on **Search**: clear the query box. This switches to the
**Timeline** - a chronological browse of everything you've recorded
so far, filterable by tag and date range. Create a tag from here and
assign it to an entry to see tag-filtered search work end to end.

## 4. Assistant

Go to **Assistant** and ask a plain-English question, e.g. "what is my
net worth" or "how much did I spend". The answer is generated by a
local, deterministic rule engine (no cloud LLM call - the response is
instant and works with the network disabled) and every answer cites
the specific account/transaction/document it was derived from,
directly under the response text - that citation is the "explainable
answers" requirement from `ROADMAP.md`, not a generic disclaimer.

## 5. Security

Go to **Security** to see the audit log: your earlier unlock is
already listed. Wait 5 minutes without touching bottom-nav (or just
trust the code) and the app auto-locks - `AutoLockPolicy`'s idle
timeout. Unlock again and confirm a second `APP_UNLOCKED` entry
appears.

## 6. Data (export / import / backup)

Go to **Data**:

1. **Export JSON** - saves your full dataset (every entity you created
   above) to a file you choose. Open it in a text editor; it's
   human-readable, pretty-printed JSON.
2. **Export CSV** - saves transactions only, in spreadsheet-friendly
   form (if you have none yet, this produces a header-only file - not
   an error).
3. **Export encrypted backup** - saves the whole SQLCipher-encrypted
   database file. Opening it in a plain text/hex editor shows opaque
   ciphertext, not your data.
4. **Restore backup** - pick the file from step 3. You'll see an
   explicit confirmation dialog (this is destructive and closes the
   app) before it proceeds; after confirming, the app exits. Reopen it
   and unlock again - your data is unchanged, proving the round trip.

Check **Security**'s audit log one more time: `DATA_EXPORTED` (x2) and
`BACKUP_EXPORTED`/`BACKUP_RESTORED` entries are now present alongside
the unlock entries - every data operation is audited.

## 7. Back to Dashboard

Return to **Dashboard**. Net Worth, Cash Flow, and Recent Activity now
reflect everything entered above, live - there is no "refresh" action
anywhere in the app; every screen observes the database directly via
`Flow`.

## What this walkthrough proves

- The app is fully usable with the network disabled at every step.
- Every screen reads from and writes to one encrypted local database -
  nothing is mocked or screen-specific.
- Search, the AI assistant, and the audit log all stay consistent with
  whatever you do on any other screen, live.
